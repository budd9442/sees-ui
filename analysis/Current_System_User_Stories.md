## System Limitations Identified

### Manual Processes
- **Physical Form Registration**: Students must fill out physical forms for module registration with manual prerequisite checking
- **Manual Eligibility Checks**: Staff and HOD manually verify student graduation eligibility and class standing
- **Manual Statistics Generation**: All performance statistics and trends are generated manually
- **Manual Module Statistics**: Academic staff manually compile module performance data
- **Manual Prerequisite Validation**: Staff manually check module prerequisites against degree guidebook
- **Manual Class Standing Calculation**: Academic staff manually calculate First/Second Class eligibility

### Fragmented Systems
- **Faculty Information System (FIS)**: Used for grade uploads, result viewing, and GPA checking
- **Learning Management System (LMS)**: Used only for degree program and pathway selection
- **Physical Documentation**: Degree guidebook for module information, prerequisites, and requirements
- **Email/Physical Meetings**: Primary communication method between students and advisors
- **Manual Record Keeping**: Student academic records maintained in multiple physical and digital systems

### Communication Challenges
- **Limited Communication Channels**: Only physical meetings and email available
- **No Real-time Updates**: Students must manually check systems for updates
- **Fragmented Information**: Information scattered across multiple systems and physical documents
- **No Automated Notifications**: Students not automatically notified of grade releases or academic status changes

### Efficiency Issues
- **Time-consuming Processes**: Manual form filling, eligibility checking, and statistics generation
- **Error-prone Operations**: Manual data entry, calculations, and prerequisite validation
- **Limited Accessibility**: Physical forms and meetings restrict access and flexibility
- **No Integration**: Systems operate independently without data sharing or automated workflows
- **Manual Capacity Management**: No automated module capacity and scheduling conflict resolution

## Pain Points Summary

1. **Manual Module Registration**: Students must fill physical forms instead of online registration with automated prerequisite checking
2. **Fragmented System Access**: Multiple separate systems (FIS, LMS) with no integration or data sharing
3. **Manual Eligibility Verification**: Staff must manually check graduation requirements and class standing
4. **Manual Statistics Generation**: All reporting requires manual data compilation and analysis
5. **Limited Communication**: Only physical meetings and email for advisor-student communication
6. **No Real-time Updates**: Students must manually check multiple systems for information
7. **Physical Documentation Dependency**: Reliance on printed degree guidebooks for module information
8. **No Automated Notifications**: Students are not automatically notified of grade releases or updates
9. **Manual Prerequisite Validation**: Error-prone manual checking of module prerequisites and co-requisites
10. **No Automated Class Standing**: Manual calculation of First/Second Class eligibility
11. **Manual Capacity Management**: No automated handling of module capacity and scheduling conflicts
12. **No Integration**: Systems operate independently without automated workflows
